网络安全

一星期实战总结（四）

一直以为不可能会完成的任务，怀疑着自己的能力。 只要压力够大趋使着你，你都可以做的很优秀。
对于这星期的成果，我很满意，在此记录一下我所做过的，还有我所想的。

（起初创立这博客只是想记录我的成长记录，经过一年写了很多文章，现在要写一些比较敏感的东西，所以也不会写太详细，自己能看得懂足已，第一，这个博客我从来没发过到某些圈子里承认是我自己的，其二，或许别人可以通过各种手段查找到此博客，期三，我没留下任何的联系方式还有留言，就此写下我的经历和成长。）

其实这批列表在我手里放了一个多月，之前通过Struts爆出来的漏洞搞了一个，只是停留在边界上那时候也花了挺多时间，后来被管理员发现修洞删Shell。
因为某地区实在搞得没有思路，所以换下目标，这批列表大概有将近30多个站，很多都是国外比较出名的，可以说是世界顶尖科技，其实这里我不窃取资料这些对我来说不感兴趣，其次，我的目标也不是这些资料。

初次试探

对于这30多个站点，我的任务只需拿下一个足已，所以刚开始通过一些批量扫描，没有对一个站点而进行深入看能不能找到软柿子，然后写了一个Python小脚本，用第三方工具查找所有子域名：

然后批量扫敏感信息泄漏，各种扫描都扔，后来结果不是很理想，太小看这些国外站了，其实也是在我意料之中。后来爆了Struts漏洞，这里可以看一星期实战总结（二）所遇到的问题，之后就放弃暂时没搞。

重整旗鼓

这次找了个日本的企业，像单刀直入肯定是行不通或许真有那样的能力， 搞进去也没有想要的东西吧，鬼使神差的让我找到了某个国家的站点，其实是这家公司的另外的分公司，主要做高端技术。

从二级域名下手，发现CRM，自己的目标就是销售记录，用的某个比较出名的CRM系统。通过一翻查找之后，发现了前几年爆出来过能够GetShell，不管怎么样，先去尝试，不管多老的洞，去尝试总有一份机会，不尝试一份都没有。

利用GetShell代码直接取得Shell,眼看任务就要完成了，这里上传了一句话木马访问正常，也能显示phpinfo,就是无法通过菜刀连接，实在没办法后来只能用大马代替好了，用了习科大马100多kb，无法猜测出地址，所以只能换更小的马，还是因为之前自己搞站的时候获取国外的Shell，还是挺好用的。只不过功能太简单。 更奇葩的是，上传一些东西会被杀，这里我猜测服务器或话有杀软。后来使用了一款b374K马。
Download:https://github.com/b374k/b374k.git,完美解决。

已获得WebShell,然后思路是这样的：WebShell -> 获取SQLServer密码 -> 内网Socks5 -> 导出数据 -> 完成目标 -> 人生巅峰。

其实后来是这样子的：

WebShell -> 获取源码 -> 源码被加密 -> 猝...

花了大量时间 -> 解密 -> 死活找不到mssql连接密码 -> 猝...

花了大量时间 -> 找到密码 -> Socks5无法连接3389 -> 无法连接数据库 -> 猝死...

花了大量时间 -> 各种端口转发 -> 分析网络连接 -> 发现mssql连接内网另外一台机器 -> 猝...

后来用Cobal Strike生成木马执行，通过Mimikatz抓取密码为空，HashDump获取密码也是为空，因为Windows 2008 R2服务器之后是不可以直接抓取hash需要修改注册表，后来改完之后再抓还是不成功就不知道是为什么了。

第二天继续的时候发现WebShell和木马已经被管理员清除了，很害怕管理员修复了漏洞然后此次花了大量时间找口子就要Get Over了，还好没修复，重新传了Shell也通过木马成功上线，再次尝试通过Mimikatz直接获取了域管理员。真是运气真好。

net view获取同段机器,这个段的机器都挺重要的包括ERP，各种数据服务器，然后成果是这样，花了大量的时间：
图太大没有办法完全截完。

其间还遇到了各种各样的问题使用了如何把获取到的东西如何传输出来，以及winrar命令、隐藏自己。

后续

经过一个星期之后，说之前拿到的某样东西没用什么样，所以又重新开刀搞起，过了一个多星期多多少少细节已记不清楚了。

大概思路是，为了能获取某台系统的数据，必需拿到数据库，因为有域控，所以拿下内网任何一台机器也不在话下，最主要的是只能通过Cobalt Strike来操作，这玩意传输数据太慢了，没有像Windows远控一样强大可以控制屏幕，其实是有vnc功能的只是需要安装，不想被发现所以就换了一条思路。

买了台vps搞了个远控，发现远控种上去远程屏幕是黑屏，测试了好几台也是如此，怀疑是远控比较久远只能支持Windows2003以及Win7界面，对win2008 r2只能以黑屏显示。所以来了挺长时间时间相接的远控。连之。
但又发现了新问题，因为屏幕被锁定了需要按Ctrl+Alt+Del才能进入到登录界面，后来找了远程修改组策略，主要是先修改本地的组策略，最后更新在目标服务器上。Links:https://wenku.baidu.com/view/33c85e0af12d2af90242e63e.html。可能是加入了域的原因和本地复现有所不同。

大概等到了晚上8点的时候，找了一台没业务系统的服务器，重启之。后来就进入到了各种Mstsc连接机器。
看数据库有100G之多，找些重点想要的数据库脱了出来，经过本地还原。发现数据库表有上千个之多，又要找到很多关联性的数据库。这时候我已经放弃了。

再次进到内容慢慢找，发现文件服务器，脱。。。完事。

总结:此时渗透过程运气和技术以及耐心是离不开的，也让我知道了所谓的渗透是什么，真正的一次渗透又该如何。
也让我往以后学习的道路更加清晰。
一次APT攻击是不会被人发现的，也不会被对方猜出攻击者是谁，有什么目的。
