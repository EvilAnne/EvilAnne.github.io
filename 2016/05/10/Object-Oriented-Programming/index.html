Object Oriented Programming

面向对象东西太多，之前学过老是会忘记里面的一些东西，借此有博客也开始动手写笔记。

实例化对象

  • 类的定义以class关键字开始
  • 定义类的属性
  • 定义类的方法
  • 实例化类的对象
  • 使用对象的属性和方法
 类实例化为对象时使用new关键字，new之后紧跟类的名称和一对括号。
调用对象的某个方法使用的语法是->符号，后面跟着方法的名称和一对括号

函数

构造函数

默认的构造函数，在对象被实例化的时候自动调用
构造函数通常用于初始化对象的属性值
$this是php里面的伪变量，表示对象自身

析构函数

析构函数，用于清理程序中使用的系统资源，比如释放打开的文件等等
析构函数在该对象不会再被使用的情况下自动调用

tostring、invoke()

  • 当对象被当做String使用时，__tostring()会被自动调用
  • 当对象被当成方法调用时，__invoke()会被自动调用

方法重载

__call()

  • 当对象访问不存在的方法名称时，__call()方法会被自动调用

__callStatic()

属性重载

set()、get()、isset() 、unset()

  • 在给不可访问属性赋值时，__set() 会被调用。

  • 读取不可访问属性的值时，__get() 会被调用。

  • 当对不可访问属性调用 isset() 和empty()时，__isset() 会被调用。

  • 当对不可访问属性调用 unset() 时，__unset() 会被调用

继承

  • PHP中类不允许同时继承多个父类，也就是extends后面只能跟一个父类名称，这个特性被称为PHP的单继承特性

访问控制

public

  • public的类成员可以被自身、子类和其他类访问

protected

  • protected的类成员只能被自身和子类访问

private

  • private的类成员只能被自身访问

静态成员

  • 静态属性用于保存类的公有数据
  • 静态方法里面只能访问静态属性
  • 静态成员不需要实例化对象就可以访问
  • 类内部，可以通过self或者static关键字访问自身的静态成员
  • 可以通过parent关键字访问父类的静态成员
  • 可以通过类名称在外部访问类的静态成员
  • self用于表示当前类，”::”操作符用于访问类的静态成员
  • static关键字也可以用于访问当前类的静态成员
    • parent用于表示父类，可以用于访问父类的静态成员

重写和Final

  • 子类中编写跟父类完全一致的方法可以完成对父类方法的重写
    • 对于不想被任何类继承的类可以在class之前添加final关键字
    • 对于不想被子类重写（overwrite, 修改）的方法，可以在方法定义前面添加final关键字

如果父类方法不想被重写，前面加final

接口

  • 接口里面的方法没有具体的实现
  • 实现了某个接口的类必须提供接口中定义的方法
  • 不能用接口创建对象，但是能够判断某个对象是否实现了某个接口
  • 接口可以继承接口（interface extends interface）
  • 接口中定义的所有方法都必须是公有，这是接口的特性。

抽象类

  • 抽象类允许类里面的部分方法暂时没有具体实现，这些方法我们成为抽象方法
  • 一旦类里面有抽象方法，这个类就必须是抽象类
  • 抽象类跟接口一样，不能直接实例化为对象
