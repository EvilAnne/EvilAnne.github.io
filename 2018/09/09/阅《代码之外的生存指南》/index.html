如何面试？

摘抄：

花时间研究这家公司、找出这个公司有哪些开发人员在写博客，并且在他们博客下留言。

从期待加入的公司里找到联系人 -> 与该联系人建立紧密关系 -> 获得个人推荐 -> 面试

在还没开始离职之前就需要花时间去建立这些关系。

为了获得面试经验，你也要去面试，获取现在面试所考的问题以好准备针对性复习。



列出你想去工作的公司，以及你认识的这些公司的人。

制定计划至少去认识这些公司中的一位员工，并与之建立联系。

我的思考：

略

下面是我以面试官来看待面试的问题：

一年下来看的简历和面试过的人没有100也有50了，招聘信息岗位和实际用人单位的要求不符合，
有时候各类招聘要求都写的非常高，但实际要求并非像招聘网站所写的那样，很多都是互相Copy以示友好。

想知道你做过哪些有趣的事情或者有难度的事情来证明你的技术能力。

你学过哪些东西学的如何还有你的知识面，主要考察你的学习能力，遇到问题如何去解决还是放弃。

还有一个就算你当前的能力不足，你要有自信在试用期内成长和进步，这是给面试者尝试的机会。

专业深入

摘抄：

在一个专业方向上拥有专长，专业程序需要深。

思考问题：

列出你能想到的所有专业领域，从广义的范畴将它们细化。

通过招聘网站上看行情，确定一下，专业化道路是会让自己受益，还是过度限制了自己的选择。

我的思考：

根据现有方向发展，编程基本功深入发展

如何写简历？

看过大量关于安全人员的简历，写的都不太让我满意有也只是非常少数。

我想在简历看到的而是别人简历中没有的东西，如果你的简历都差不多所以我会觉得没有什么突出点。

思考问题：

不管你现在是否正在找工作，把你的简历给一些招聘人员，询问他们的意见。

如何在简历中体现自己的亮点能让在海量的简历库中发光。

其实自己在写简历这一块也不是很丰富，看的时候比较多大概哪些痛点或者能让人对你这人感兴趣觉得能胜任此工作，

只要面试者表现的足够自信的前提下。

  • 自我营销

个人博客

思考问题：

如果你还没有博客，开通一个，你会专注在哪些主题上？

开通博客也是自己的个人品牌，在简历上也可以贴了，在博客中有让人感兴趣或者有技术含量的文章以及你的学习过程
都会被看到。

社交媒体

摘抄：

你需要学会如何通过分享的内容和分享的方式塑造自己的形象。

保持活跃度，每天都要管理和维护自己的Twitter、Facebook或其它媒体

分享你的学习过程认识更多的人，也帮助哪些需要帮助的人，说不定别人哪天就超过你而且反过来指导着你。

除此之外你发表的文章也可以贴在简历上以证明你的技术能力。

百折不挠，越挫越勇

摘抄：

必须学会不在乎别人怎么想，你必须学会如何无惧自己被别人看作傻瓜。

事实上，万物都在变化之中。随着时间流逝，原本让你不适的东西你最终都会应对自如。你需要足够的时间和意愿去克服尴尬，
直到它们不再让你感到局促不安。

被看作傻瓜又如何，你必须要学会收起自己脆弱的自尊心，勇敢走出去。

习惯些辱骂是件好事，因为即使再完美的作品也会遭人抨击。你永远不可能取悦每一个人。

思考问题：

让自己勇敢起来，走出去做一些令你害怕的事情，大小都不要紧，迫使自己处于不自在的环境，并且提醒自己，
这没什么大不了的。

  • 学习

学习怎样学习

摘抄：

如果你能将自己所学的东西教给别人，不仅能记住，还能理解得更深刻。

以玩为学习目的，这种自然的探索和好奇心帮助我们轻易举地记住了那么多的信息。

十步学习法

摘抄：

步学习法的基本思想就是：要对自己要学的内容有个基本的了解——了解自己不知道什么就足矣。
然后利用这些信息勾勒出学习范围，即需要学习哪些内容，以及学成之后又会获得什么。

步学习法的第一部分是研究，它是一次性完成的。但是从第7步到第10步则是重复的过程，
贯穿于你的学习计划的各个模块。

第1步到第6步 这些步骤只做一次

摘抄：

    你需要集中精力完成足够多的前期调研，确保自己明确知道要学哪些内容，以及如何确认自己已达成目标。
  1. 了解全局
  2. 确定范围
  3. 定义目标
  4. 寻找资源
  5. 创建学习计划
  6. 筛选资源

  • 了解全局

    你对自己到底要学什么理解并不透彻。

    粗略浏览一下内容，但是不要在这一步上花费太多时间。这一步的目的不是掌握该主题，只是对这一主题的相关内容有一个全局的了解。

  • 确定范围

    集中精力去明确自己到低要学什么。让自己的关注点落脚到更小也更可控的范围。

    如：学习C# 掌握如何创建一个简单的控制台程序，学习Linux 了解如何安装设置，以及如何其它他的基本功能。

    找出自己的学习范围。同时也可以借助自己的学习理由来决定学习的范围。

    保持专注，你一次只能学一样东西。可以稍后再回头学习别的分支领域，但目前而言，选择一个专注而聚焦的范围，潜心学习。

    明确学习时间以及学到什么样的程度。

  • 定义目标

    如果不知道成功是什么样子，很难找准目标，也很难知道自己什么时候真正达到目标。在尝试学习任何东西之前，你都应该在脑海里中清晰地描绘出成功的样子。

    这一步的目标是形成一份简明清晰的陈述，勾勒出你勤奋学习后的成功图景，要列出特写的结果，或者实现自己所能达到的目标你应该能够做到的事情。

  • 寻找资源

    收集到多种多样的资源以帮助你学习，找大量的资源：图书、文章、视频、源代码、在线文档等

  • 创建学习计划

    创造自己的学习路径。

  • 筛选资源

    回到第4步，你收集了与研究主题相关的所有资源。在第5步中你使用这些资源制订了自己的个人学习计划。对资源进行筛选，找出有价值的几项来帮你实现自己的目标。

第7步到第10步 循环重复摘抄

摘抄：

学习—实践—掌握—教授（LDLT）

你从掌握恰到好处可以开始的基础知识开始，然后通过操作来学习，同时也通过自我探索收集问题。
之后你掌握了足够多的有用知识，你能将自己学到的教给他人，以此来弥补自己在学习过程中的不足。
  1. 开始学习，浅尝辄止
  2. 动手操作，边玩边学
  3. 全面掌握，学以致用
  4. 乐为人师，融会贯通

  • 开始学习，浅尝辄止

    专注于掌握自己所需的、能在下一步动手操作的最小量的知识。足以让你对自己要做什么有基本认识。

  • 动手操作，边玩边学

    你无需提前了解全部的内容，你要做的首要的一件事情就是亲自操作和亲身体验。通过探索和实践进行学习，在操作的过程中，你的大脑自然地产生各种问题。

    它们是如何工作的？如果我这么做会发生什么？我该如何解决这个问题？

  • 全面掌握，学以致用

    好奇心是学习特别是自学的重要组成部分。

    你要先利用先前收集到的所有资料，进行深入学习。

    阅读文字、观看视频、与人交流以，这能让你沉浸在学习资料中，尽可能地汲取知识。

  • 乐为人师，融会贯通

    大多数人都不敢为人师，思考自己知道的东西是否值得教给别人的时候，很容易陷入自我怀疑之中，但是如果你想深入地掌握一门学问，想对这门学问做到融会贯通,那么必须要到 好为人师。

    走出自己的舒适区，将自己学到的知识教给别人。

    花时间将自己学到的东西从大脑中提取出来，以别人能够理解的方式组织起来，在经历了整个这个过程之后你会发现，有很多你以为自己明白了的知识点，其实并没有摸透。

    没人愿意跟一个对自己讲的内容豪无低气的人学习。
